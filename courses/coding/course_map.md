# 学生编程水平评估追踪表（目标：信息学奥赛）

**初始状态说明**
- 学生年级：小学四年级（~10 岁）
- 编程经验：略微接触过（可能玩过 Scratch 或 Blockly 编程游戏）
- 数学基础：校内 90 分，奥数初步接触中
- 目标：信息学奥林匹克竞赛（NOI）核心赛道
- 掌握程度标签（Level）：
  - Level 0：未接触
  - Level 1：理解概念（能解释，不能独立编码）
  - Level 2：框架完成（能在提示下完成代码）
  - Level 3：独立正确（能独立写出正确代码）
  - Level 4：优化版本（代码正确且高效，边界完善）
  - Level 5：举一反三（能解决变体问题）

**使用方式**：根据每次编程练习的表现，修改对应单元格的掌握 Level。

## 首次评估说明

由于学生编程经验有限（仅"略微接触过"），**首次课程将作为全面诊断课**：

1. **编程思维测试**（通过 Scratch/Blockly 风格的问题）：
   - 能否理解"重复 5 次"这样的循环概念？
   - 能否将"找到最大的数"分解成步骤？
   - 条件判断的直觉如何？

2. **打字与工具熟悉度**：
   - 键盘熟悉程度？打字速度？
   - 是否知道如何安装/运行一个程序？

3. **兴趣与动机探索**：
   - 最喜欢什么类型的游戏/应用？（找到编程兴趣点）
   - 对"做出一个能运行的程序"有多期待？

4. **数学前置知识检查**：
   - 整数运算、比较大小熟练度
   - 对"规律"和"模式"的敏感度

根据诊断结果，**推荐从阶段一或阶段二的适当位置开始教学**。对于 10 岁学生，建议从阶段一起步，用 Scratch 或简单 C++ 程序培养编程兴趣和信心。
